## 📖 Описание

**Linear Algebra AI Detector** — это десктопное приложение для преподавателей российских технических вузов, определяющее вероятность использования искусственного интеллекта при написании учебных работ по **линейной алгебре** с учётом российских математических стандартов.

Приложение работает **без доступа к интернету** и не отправляет ваши данные на внешние серверы. Все вычисления производятся локально на вашем компьютере.

### Зачем это нужно?

Современные языковые модели (ChatGPT, GPT-4, Claude и др.) часто используют западные математические обозначения (`tan` вместо `tg`, `eigenvalue` вместо «собственное значение»), а также шаблонные фразы и канцелярские обороты. Приложение анализирует текст и выявляет эти признаки, помогая преподавателям объективно оценивать работы студентов.

## 🏷️ База маркеров

### Что такое маркер?

Маркер — это регулярное выражение, которое ищет в тексте признаки использования ИИ. Каждый маркер имеет:

- **ID** — уникальный идентификатор (например, `MATH-001`)
- **Pattern** — регулярное выражение для поиска
- **Weight** — вес от 1 до 10 (влияет на итоговую вероятность)
- **Category** — категория (тригонометрия, линейная алгебра, шаблоны и т.д.)
- **Description** — описание проблемы
- **Severity** — серьёзность (low, medium, high, critical)

### Категории маркеров

| Категория | Описание | Пример |
|-----------|----------|--------|
| `trigonometry` | Тригонометрические функции | `tan` вместо `tg` |
| `linear_algebra` | Термины линейной алгебры | `eigenvalue` вместо «собственное значение» |
| `template` | Шаблонные фразы | «следует отметить», «в заключение» |
| `lexical` | Лексические особенности | «то есть» вместо «т.е.» |
| `formatting` | Форматирование | Отсутствие знака умножения |
| `exponential` | Экспоненты | `exp()` вместо `e^` |
| `custom` | Пользовательские маркеры | Добавляются вручную |

### Управление маркерами

1. Нажмите кнопку **«✏️ Управление маркерами»**
2. Откроется редактор со списком всех маркеров
3. **Добавление:** Заполните поля и нажмите «➕ Добавить»
4. **Редактирование:** Выберите маркер, измените поля, нажмите «💾 Обновить»
5. **Удаление:** Выберите маркер, нажмите «🗑 Удалить»
6. **Включение/отключение:** Используйте чекбокс «Активен»

### Пример добавления пользовательского маркера

```json
{
  "id": "CUSTOM-001",
  "pattern": "\\bнейросеть\\b",
  "weight": 4,
  "category": "custom",
  "description": "Упоминание нейросетей",
  "severity": "low"
}
```

## 🧮 Алгоритм расчёта

### Формула

```
Общий_балл = Σ(вес_маркера × min(3, количество_вхождений))
Максимальный_порог = Σ(вес_маркера × 3) для всех активных маркеров
Итоговая_вероятность_ИИ = min(100, (Общий_балл / Максимальный_порог) × 100)
```

### Почему ограничение 3 вхождения?

Исследования показывают, что даже однократное использование характерного для ИИ маркера — сильный признак. Однако многократное повторение одного и того же маркера не увеличивает вероятность линейно. Поэтому каждое вхождение после третьего не учитывается.

### Пример расчёта

| Маркер | Вес | Найдено | Очки |
|--------|-----|---------|------|
| `tan()` вместо `tg()` | 8 | 4 → 3 | 24 |
| `eigenvalue` | 8 | 2 | 16 |
| «следует отметить» | 5 | 1 | 5 |
| **Итого** | | | **45** |

Максимальный порог для этих маркеров: (8×3) + (8×3) + (5×3) = 63

Вероятность = 45 / 63 × 100 = **71.4%**

## 🐛 Устранение неполадок

### Ошибка: «Не удалось загрузить файл»

**Причина:** Формат файла не поддерживается или файл повреждён.

**Решение:**
1. Убедитесь, что формат файла в списке поддерживаемых
2. Попробуйте открыть файл в другой программе
3. Сконвертируйте файл в TXT или DOCX

### Ошибка: «Ошибка инициализации базы данных»

**Причина:** Нет прав на запись в папку `%LOCALAPPDATA%`.

**Решение:**
1. Запустите приложение от имени администратора
2. Проверьте свободное место на диске
3. Вручную создайте папку `%LOCALAPPDATA%\BRAgInc\LinearAlgebraDetector\`

### Ошибка: «Не удалось найти маркеры»

**Причина:** База маркеров не загрузилась.

**Решение:**
1. Убедитесь, что файл `markers.json` находится в папке `Resources/`
2. Перезапустите приложение
3. Нажмите «Сбросить базу маркеров» в редакторе

### Проблема: Медленная работа

**Причина:** Очень большой текст (>10 МБ) или слабый компьютер.

**Решение:**
1. Разбейте текст на несколько частей
2. Используйте формат TXT вместо PDF/DOCX
3. Закройте другие ресурсоёмкие приложения

### Проблема: Не отображается подсветка

**Причина:** RichTextBox не поддерживает HTML.

**Решение:**
1. Используйте экспорт в HTML и откройте в браузере
2. Обновите .NET до последней версии

## ❓ Часто задаваемые вопросы

### 1. Нужен ли интернет для работы?

**Нет.** Приложение полностью автономно и не требует подключения к интернету.

### 2. Отправляются ли мои данные на сервер?

**Нет.** Все данные остаются на вашем компьютере. Приложение не собирает и не передаёт никакую информацию.

### 3. Могу ли я добавить свои маркеры?

**Да.** Используйте редактор маркеров (кнопка «✏️ Управление маркерами»).

### 4. Как сбросить базу маркеров к заводским настройкам?

Удалите файл `%LOCALAPPDATA%\BRAgInc\LinearAlgebraDetector\markers.db` и перезапустите приложение.

### 5. Работает ли приложение на Linux/macOS?

Код является кроссплатформенным. Для сборки под Linux/macOS измените `RuntimeIdentifier` в `.csproj` на `linux-x64` или `osx-x64`.

### 6. Можно ли использовать приложение для других дисциплин?

Да, добавив соответствующие маркеры через редактор. База данных гибкая и расширяемая.

### 7. Как часто обновляется база маркеров?

База маркеров обновляется при выходе новых версий приложения. Вы также можете добавлять маркеры вручную.

### 8. Поддерживает ли приложение другие языки?

Интерфейс только на русском. Анализ текста работает с любым языком, но маркеры оптимизированы для русского и английского.
